# SmartBrowser Update Report (2026-03-22)

## Scope

This update focused on reliability, planning quality, API usage efficiency, browser profile UX, UI readability, and repository hygiene.

## Implemented Changes

### 1. Browser Profile Dropdown in Browser Settings

- Added profile discovery and resolution utilities in `src/browser/profile_utils.py`.
- Added a new `Default Browser Profile` dropdown and `Refresh Profiles` button in `src/webui/components/browser_settings_tab.py`.
- Supported presets:
  - `Chrome - Default` and detected `Chrome - Profile N`
  - `Edge - Default` and detected `Edge - Profile N`
- Dropdown selection now auto-populates browser binary and user-data-dir (manual values still override).
- Profile selection is wired into both execution paths:
  - Browser agent (`src/webui/components/browser_use_agent_tab.py`)
  - Deep research agent (`src/webui/components/deep_research_agent_tab.py`)

### 2. Better Planning Before Execution

- Added `Force Task Planning` setting in `src/webui/components/agent_settings_tab.py`.
- If enabled and no planner model is configured, the main LLM is reused as planner fallback.
- Added task execution guardrails in `src/webui/components/browser_use_agent_tab.py` to instruct:
  - plan first,
  - avoid guessing URLs,
  - prefer verified search results,
  - avoid repeated failed URLs.

### 3. API Request Conservation and Rate-Limit Reduction

- Removed expensive post-task site-description LLM call in `src/webui/components/browser_use_agent_tab.py`.
  - This eliminated an extra completion request after every task.
- Added fail-fast repeated-URL failure guard in `src/agent/browser_use/browser_use_agent.py`.
  - Prevents excessive retry loops on the same failing URL.
- Added query normalization + deduplication in deep research search tool (`src/agent/deep_research/deep_research_agent.py`).
  - Reduces repeated browser tasks and token spend.

### 4. SmartBrowser Title Visibility Fix

- Updated header title CSS in `src/webui/interface.py`:
  - Added visible fallback color and text shadow.
  - Kept gradient text only when browser supports clip behavior.

### 5. File/PDF Task Reliability

- Added upload-path state wiring in `src/webui/components/browser_use_agent_tab.py`.
- Added local file context extraction for text and PDF uploads.
- Added `pypdf` dependency in `requirements.txt`.
- Hardened upload action in `src/controller/custom_controller.py` for empty available-file cases.
